# AgoraMesh V1.0 개발 로드맵 (수정안)

> 상태: 수정 초안 (Revised Draft)  
> 목표: 중앙 운영자 없는 분산 자치 커뮤니티의 최소 실행 가능 제품(MVP)  
> 버전 정책: `v0.x` = milestone preview, `v1.0` = 첫 안정 릴리스, `v1.1+` = 선택적 확장

---

## 1. 기존 로드맵의 문제점 요약

1. **V1.0 범위 과대**: DM, 검색 인덱서, 웹 게이트웨이, 연합 피드 고급 기능, 짤팩 고급 기능까지 V1.0 필수로 포함되어 실제 개발 일정과 검증 부담이 비현실적이었다.
2. **Phase 0 과도 동결**: 프로토타입 구현 전에 gossip 방식, 저장소 구조, 투표권 계산식, media_ref 세부 필드, discovery 방식까지 frozen 처리하려 했다.
3. **UX 검증 시점 지연**: 클라이언트 UI와 관리자 대시보드를 후반 Phase에 배치해, 프로토콜 설계가 실제 사용성과 맞는지 초기에 검증할 수 없었다.
4. **시빌 방지 과도 추상화**: 전역 평판 그래프를 V1.0에서 구현하려 해 설명 가능성과 개발 현실성이 떨어졌다.
5. **공식 인프라 오해 가능성**: 릴레이/검색/게이트웨이 "기능 제공" 표현이 메인테이너가 운영하는 것처럼 읽힐 수 있었다.
6. **보안/테스트/운영 산출물 부족**: threat model, protocol conformance test, 릴리즈 서명, 백업/이주, 악용 시뮬레이션이 체계적으로 포함되지 않았다.

---

## 2. 수정된 V1.0 정의

V1.0은 "완성형 분산 커뮤니티 전체"가 아니라, **일반 유저와 카테고리 관리자가 실제로 사용할 수 있는 최소 자치 텍스트 커뮤니티 제품**이다.

- 메인테이너는 프로토콜/클라이언트 개발자일 뿐, 어떤 인프라도 공식 운영하지 않는다.
- 텍스트 게시글·댓글·카테고리·헌장·선출형 관리자·신고·숨김·항소를 완전히 동작하게 한다.
- 이미지는 `media_ref`로 분리하고, 최소한의 `media-node` 레퍼런스 구현만 포함한다.
- 초반부터 최소 UI/TUI를 만들어 프로토콜 설계의 사용성을 검증한다.
- DM, 검색 인덱서, 웹 게이트웨이, 고급 연합 피드, 고급 짤팩은 V1.1 이후로 미룬다.

## 6. 각 Phase별 목표, 작업, 완료 조건

### Phase 0. 원칙 / 위협모델 / 문서 뼈대

**목표**: 구현 전 반드시 고정할 것만 고정하고, 나머지는 provisional로 둔다.

**고정 (frozen)**:
- 불변 원칙 (중앙 서버/릴레이/검색/게이트웨이/카테고리 목록/미디어 노드/추천 없음, 기본 접속 노드 내장 금지)
- 위협 모델
- 책임 경계 (메인테이너, 사용자, 카테고리 관리자, 텍스트 노드 운영자, 미디어 노드 운영자)
- 문서 구조
- 메인테이너가 운영하지 않는 항목 명시

**Provisional (이후 동결)**:
- 객체 스키마
- gossip 방식
- 저장소 구조
- 투표권 계산식
- `media_ref` 세부 필드
- discovery 방식

**작업**:
| # | 작업 | 산출물 |
|---|---|---|
| 0.1 | 불변 원칙 문서 작성 | `docs/protocol-overview.md` |
| 0.2 | 메인테이너 포지션 문서 작성 | `docs/maintainer-position.md` |
| 0.3 | 위협 모델 초안 작성 | `docs/threat-model.md` |
| 0.4 | 문서 구조 및 작성 규칙 확정 | `docs/README.md` 내부 링크 구조 |
| 0.5 | 법률 검토용 문구 확정 | `README.md`, `docs/legal-notes.md` |
| 0.6 | 카테고리 헌장 템플릿 초안 | `docs/charter-template.md` |
| 0.7 | 카테고리 번들 개념 초안 | `docs/category-bundle.md` |
| 0.8 | 릴리즈 서명 정책 초안 | `docs/release-signing.md` |

**완료 조건**:
- `docs/maintainer-position.md`가 메인테이너가 운영하지 않는 항목을 명시한다.
- `docs/threat-model.md`가 최소 5개 공격 시나리오를 포함한다.
- 문서 구조 목록이 확정되고 README에 링크된다.

- 동일한 객체가 동일한 hash를 가진다.

### Phase 3. 카테고리 헌장 + 관리자 + 신고/숨김/항소

**목표**: 카테고리가 헌장 기반으로 자치할 수 있는 핵심 루프를 완성한다.

**작업**:
| # | 작업 | 핵심 요구사항 |
|---|---|---|
| 3.1 | `category_charter` 객체 | 주제, 금지 행위, 관리자 수/임기/선출/해임, 투표권 조건, 긴급 숨김 조건, 항소 절차, 포크 절차, 미디어 모드 |
| 3.2 | 투표 객체 | `vote` 타입, 종류, 만료, 가중치 규칙 참조 |
| 3.3 | 관리자 선출/해임 | 투표 발의, 집행, 공개 로그 |
| 3.4 | `report` 객체 | 사유 코드, 대상 참조, 증거, 서명 |
| 3.5 | `report_bundle` 객체 | 같은 게시물/댓글/media_ref/유저/노드/사유/시간대 자동 묶음 |
| 3.6 | `moderation_action` 객체 | 숨김/차단/기각/항소 가능 처리, 관리자 서명, 기간 |
| 3.7 | `tombstone` 객체 | 숨김/전파중단 상태, 사유, 원래 객체 참조 |
| 3.8 | 항소 흐름 | 항소 객체 생성, 투표 연결, 결과 집행 |
| 3.9 | 공개 관리 로그 | `audit_log` 객체, 모든 관리자 조치 서명 기록 |
| 3.10 | 투표권 계산 (provisional) | 활동 기간 기반, 1계정 1표 금지 방지, 전역 점수 없음, 개인정보 없음 |

**완료 조건**:
- 한 카테고리에서 관리자 선출 투표가 성사되고 결과가 `audit_log`에 남는다.
- 신고 5건이 같은 `post`에 쌓이면 `report_bundle`로 묶인다.
- 긴급 숨김 후 항소 절차가 자동 트리거된다.
- 오래된 헌장의 재전파가 방지된다.

### Phase 4. media_ref + 최소 media-node

**목표**: 이미지를 코어 프로토콜에서 완전히 분리한다.

**작업**:
| # | 작업 | 핵심 요구사항 |
|---|---|---|
| 4.1 | `media_ref` 객체 | 해시, 썸네일 참조, origin_node, TTL, declared_mime/size |
| 4.2 | `media_node_manifest` 객체 | 노드 키, 정책, 허용 미디어, report_endpoint, compliance_statement |
| 4.3 | 게시글 내 media_ref 첨부 | 텍스트 노드가 이미지 원본을 요청하지 않음 |
| 4.4 | 최소 media-node 구현 | 업로드, EXIF 제거, 썸네일, TTL, 신고 접수 |
| 4.5 | media-node 기본 보안값 | 영상 off, 파일 off, SVG 금지, 공개 검색 off, 2MB 제한, JPG/PNG/WebP, EXIF 제거 on, TTL 30일, 저장공간 상한, 일일 업로드 제한 on, 신고함 on, 캐시삭제 on |
| 4.6 | media-node 비상 모드 | 새 업로드 중단, 신고 누적 이미지 자동 숨김, 썸네일만 제공, 특정 카테고리 media_ref 제공 중지, 저장공간 초과 시 오래된 이미지 만료 |
| 4.7 | 클라이언트 이미지 보기 | 처음 보는 미디어 노드 흐림 처리, 원본 클릭 시 로드, 외부 URL 자동 요청 금지 |

**완료 조건**:
- 텍스트 노드가 이미지 원본을 저장하지 않는다.
- `media_ref`만 있는 게시글에서 텍스트 노드가 이미지를 요청하지 않는다.
- media-node가 비상 모드로 전환될 수 있다.

## 13. 기존 원칙 재확인

- 중앙 서버 없음
- 공식 릴레이 없음
- 공식 검색 인덱서 없음
- 공식 웹 게이트웨이 없음
- 공식 카테고리 목록 없음
- 공식 미디어 노드 없음
- 공식 추천 알고리즘 없음
- 기본 접속 노드 내장 금지
- 메인테이너는 프로토콜/클라이언트 개발자
- 카테고리는 헌장, 선출형 관리자, 신고, 숨김, 항소, 투표로 자치
- 이미지는 코어 프로토콜에 직접 넣지 않고 `media_ref`로 분리
- 텍스트 노드는 이미지 원본을 저장하지 않음
- 이미지 저장, 썸네일, 신고, 캐시삭제는 별도 media-node가 담당
- “삭제”보다 “숨김”, “전파 중단”, “캐시 삭제”, “로컬 표시 제한” 표현 사용
